# Standardized Receipt Template

## Overview

All receipts generated by the POS system now follow a **single standardized template** defined in `/src/app/components/ThermalReceipt.tsx`. This ensures consistency and professionalism across all transactions.

## Standard Template Specifications

### Business Information
- **Restaurant Name**: N&Ns RESTAURANT
- **Address**: 123 Restaurant Ave., Manila, PH
- **Phone**: Tel: (02) 8123-4567
- **Receipt Title**: OFFICIAL RECEIPT

### Layout Structure

1. **Header Section**
   - Restaurant name (bold, letter-spaced)
   - Complete address
   - Contact telephone
   - Dashed separator line
   - "OFFICIAL RECEIPT" label

2. **Transaction Details**
   - Receipt ID (if available)
   - Payment ID (if available)
   - Order Number (6-digit format)
   - Date (YYYY-MM-DD format)
   - Time (HH:MM format)
   - Customer Name
   - Order Type (Dine-In, Takeout, or Mixed)
   - Table Number (if applicable)

3. **Items Section**
   - For **Mixed Orders**: Items grouped by type
     - "— DINE-IN —" header
     - Dine-in items with quantities
     - Dine-in subtotal
     - "— TAKEOUT —" header
     - Takeout items with quantities
     - Takeout subtotal
   - For **Single-Type Orders**: 
     - Items listed with quantity and total
     - Format: `{qty}x {name}` → `₱{total}`

4. **Totals Section**
   - Subtotal
   - Service Fee (1%)
   - Tax (12%)
   - Discount (if applicable, with type and percentage)
   - **Double-line separator**
   - **TOTAL** (bold)
   - Cash Received (if payment completed)
   - Change (if applicable)

5. **Footer**
   - "Thank you for dining with us!"
   - "Please come again."
   - "— N&Ns POS System —"

### Typography & Styling

- **Font Family**: Courier New, monospace (thermal receipt style)
- **Font Sizes**:
  - Header: text-sm (restaurant name)
  - Body: text-xs (all content)
  - Labels: text-gray-500 (descriptive text)
  - Values: default text color
  - Total: Bold weight
- **Spacing**: Consistent padding and margins throughout
- **Borders**: 
  - Dashed borders for section separators
  - Double border before grand total
  - Gray color (#e5e7eb / border-gray-300)

### Consistent Elements Across All Order Types

Regardless of whether the order is:
- Dine-In
- Takeout
- Mixed

The receipt maintains:
✅ Same header format
✅ Same business information
✅ Same section arrangement
✅ Same typography
✅ Same spacing and borders
✅ Same footer message
✅ Same totals calculation display

## Implementation Locations

The standardized `ThermalReceipt` component is used in:

1. **CreateOrder.tsx** - Receipt preview after takeout payment
2. **OrderList.tsx** - View receipt modal for any order
3. **Receipt.tsx** - Standalone receipt page

### Component Props

```typescript
interface ThermalReceiptProps {
  orderNumber: string;
  customerName: string;
  orderType: 'Dine-In' | 'Takeout' | 'Mixed';
  table?: string | null;
  items: ReceiptItem[];
  subtotal: number;
  serviceFee: number;
  tax: number;
  discount: number;
  discountType?: string;
  total: number;
  cashReceived?: number;
  changeGiven?: number;
  date?: string;
  time?: string;
  receiptId?: string;
  paymentId?: string;
}
```

## Print Functionality

All receipt modals include a **Print** button that triggers `window.print()`, which:
- Opens the browser's native print dialog
- Displays the receipt in printer-friendly format
- Optimized for 80mm thermal paper
- Can save as PDF or send to thermal printer
